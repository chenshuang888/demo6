  接口“位置是否恰当”的评价（重点问题在这里）

  - 现在所有组件基本都用 INCLUDE_DIRS "."（例如 services/CMakeLists.txt:1、drivers/CMakeLists.txt:1），等价于：组件目录下所有 .h 默认
    都是 public。小项目问题不大，但规模上去后容易出现：
      - 任何组件都能 include 到本不该依赖的内部头文件（架构纪律靠自觉，而不是构建系统约束）。
      - 头文件一旦被外部 include，就很难再移动/改名/拆分（API 兼容成本上升）。
  - 你已经把“语义上可能内部”的东西放进了更深的目录（例如 services/manager/、app/pages/），这方向是对的；下一步如果想让框架更“硬”，建
    议把public 头文件集中到 include/，其余走 PRIV_INCLUDE_DIRS，用构建系统把边界锁住。
