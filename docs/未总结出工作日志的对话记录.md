  5. PC 当键盘 ⭐⭐⭐⭐ 最实用

  ---
  方案设计

  路径约定

  /littlefs/apps/<id>/icon.bin        ← 菜单图标，约定文件名
  不放 assets/。理由：
  - assets/ 是给 app 自己运行时用的（前面 imgdemo 那种）；菜单图标是元信息，与 manifest.json 同级更合理
  - 跟 main.js / manifest.json 平级，filename_is_valid 已经直接接受单段名 icon.bin，零额外 C 改动
  - 卸载 app 时跟着 app 目录一起删

  推荐尺寸 / 格式

  - 24×24 或 32×32 RGB565（菜单 list item 高度大约 40-50px，留点余量）
  - 24×24 RGB565 = 1164B，32×32 = 2060B，都在单帧 64KB 上限内远远够用
  - 想透明背景就用 RGB565A8 或 ARGB8565，但增加体积

  - 强制 32×32 优点：menu list 布局规整，不用动态算 item 高度
  - 允许任意 + LVGL lv_image_set_scale 缩放：灵活但慢一点

  菜单每次进入都要 lv_image_create 读 5+ 张图，会不会卡？
  - LVGL 9 的 image decoder 有内置 cache（默认开），同一路径第二次极快

  - 不需要单独做缓存

  几个值得拍板的细节

  1. 图标 fallback 链

  - 优先：icon.bin（本 app 自带）
  - 次选：内置 LV_SYMBOL_LIST（现状）
  - 是否再加一层"manifest.json 指定 icon 字段"？不需要。约定文件名 icon.bin 比读 manifest 取字段简单一倍

  2. 图标尺寸

  强制约定 32×32？还是允许任意尺寸 LVGL 自动缩放？
  - 强制 32×32 优点：menu list 布局规整，不用动态算 item 高度
  - 允许任意 + LVGL lv_image_set_scale 缩放：灵活但慢一点
  - 推荐强制 32×32，简单高效。文档里写明，PC 工具上传时校验

  3. 颜色协调

  现在内置 SYMBOL 是 ACCENT 青绿色。自定义 .bin 是用户自己定颜色，可能跟菜单深紫主题撞色。
  - 不强求；用户自己负责美感
  - 也可以在文档里建议"主色用青绿 #06B6D4 或白"

  4. 缓存

  菜单每次进入都要 lv_image_create 读 5+ 张图，会不会卡？
  - LVGL 9 的 image decoder 有内置 cache（默认开），同一路径第二次极快
  - 32×32 RGB565 解码即拷贝 2KB，几乎零成本
  - 不需要单独做缓存

  几个值得拍板的细节

  1. 图标 fallback 链

  - 优先：icon.bin（本 app 自带）
  - 次选：内置 LV_SYMBOL_LIST（现状）
  - 是否再加一层"manifest.json 指定 icon 字段"？不需要。约定文件名 icon.bin 比读 manifest 取字段简单一倍

  2. 图标尺寸

  强制约定 32×32？还是允许任意尺寸 LVGL 自动缩放？
  - 强制 32×32 优点：menu list 布局规整，不用动态算 item 高度
  - 允许任意 + LVGL lv_image_set_scale 缩放：灵活但慢一点
  - 推荐强制 32×32，简单高效。文档里写明，PC 工具上传时校验

  3. 颜色协调

  现在内置 SYMBOL 是 ACCENT 青绿色。自定义 .bin 是用户自己定颜色，可能跟菜单深紫主题撞色。

  - 也可以在文档里建议"主色用青绿 #06B6D4 或白"

  4. 缓存

  菜单每次进入都要 lv_image_create 读 5+ 张图，会不会卡？
  - LVGL 9 的 image decoder 有内置 cache（默认开），同一路径第二次极快
  - 32×32 RGB565 解码即拷贝 2KB，几乎零成本
  - 不需要单独做缓存
