## 一句话原则

**UART 接收只负责把数据放进缓冲；由“当前模式”决定谁消费数据（Shell 解析或 OTA 写入）。**

---

## 模式路由（对话中的示意）

```c
switch (UART_GetMode()) {
    case UART_MODE_SHELL: Task_Shell_ProcessData(); break;
    case UART_MODE_OTA:   OTA_ProcessData();        break;
}
```

---

## 设计价值（为什么这是好设计）

- 一个 UART 支持多协议
- 新增模式成本低（加一个 mode + 消费者即可）
- Shell 层只负责命令解析；OTA 层是纯业务；底层驱动不变（层次清晰）

---

## 验收标准

- Shell 模式下不会误把普通输出当 OTA 数据
- OTA 模式下不会在二进制流中扫描 Shell 命令（避免误判）
- 模式切换有明确触发点（例如收到 `ota start` 后切换到 OTA 模式）
