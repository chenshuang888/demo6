## 不变式（可直接复用）

- 先跑通最小闭环（冒烟）再叠功能/优化，避免不可定位
- 只直接复用“原则与边界”，实现细节必须参数化
- 必须可观测：日志/计数/错误码/抓包等至少一种证据链

- 时序策略：超时/重试/节奏/窗口/幂等（按项目填写）
- 存储语义：写入位置、校验策略、激活/回滚策略（如适用）（按项目填写）

## 停手规则（Stop Rules）

- 上下文签名关键项不匹配（平台/版本/外设/资源/并发模型）时，禁止照搬实施步骤
- 关键参数缺失（端序/magic/分辨率/slot/payload_size/超时重试等）时，禁止给最终实现
- 缺少可执行的最小冒烟闭环（无法验收）时，禁止继续叠功能

---


## 目标

新增一个页面（例如 `PAGE_VOICE_CHAT`）时：

- 生命周期完整
- 私有状态清晰
- 与框架交互点明确
- 不引入隐式依赖

---

## 前置条件

- 已有页面框架：`PageDescriptor_t` + 页面注册机制
- 有基础 widget 库（例如标题栏绘制）
- 有 OLED 绘制 API

---

## 实施步骤（建议顺序）

1. 新建页面文件 `page_xxx.c` / `page_xxx.h`
2. 按固定顺序实现生命周期函数：
   - `Xxx_Init`（可选）
   - `Xxx_Enter`（可选）
   - `Xxx_Update`（可选）
   - `Xxx_Draw`（必须）
   - `Xxx_KeyEvent`（必须）
   - `Xxx_Exit`（可选）
3. 只在页面模块内部维护页面私有状态（`static` 变量或页面结构体）
4. 在页面表/注册处增加 descriptor：
   - `name`、函数指针、`is_initialized` 由框架管理
5. 冒烟验证：
   - 能进入页面
   - 能画标题与静态内容
   - 按键事件能触发退出/返回

---

## 验收标准

- 页面能反复进入/退出，不泄漏状态、不崩
- key_event 逻辑不会影响其它页面（事件只在当前页面消化或显式传递）
