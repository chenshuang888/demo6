## 一句话结论

**如果“显示反了、触摸也反了”，通常不是两者没对齐，而是整套坐标系一起转错了；先把全局旋转关掉恢复基线，再逐项加回 mirror/swap。**

---

## 关键判断（先分型）

### 情况 A：显示反了，触摸也反了（但点到哪就跟着反到哪）

- 结论：显示与触摸**彼此对齐**，但整体方向错
- 处理策略：优先动 **板级旋转/镜像配置**（全局策略），不要先改触摸 read_cb 的局部修修补补

### 情况 B：显示方向对了，但触摸左右/上下有一项反

- 结论：显示策略 OK，触摸映射/坐标变换有问题
- 处理策略：只动 **触摸映射**（通常在 `lvgl_port.*`），不要动显示侧

---

## 推荐落地方式（把策略集中在 `board_config.h` + `lvgl_port.*`）

对话里的落地形态是：

- `board_config.h`：定义板级旋转/镜像开关（例如 `MIRROR_X/Y`、`SWAP_XY`）
- `lvgl_port.*`：读取这些开关，统一应用到
  - 显示侧（面板 mirror/rotation 能力）
  - 触摸侧（坐标翻转/映射）

这样能保证“显示与触摸策略同源”，减少“显示改了、触摸忘了改”的复发概率。

---

## 180° 旋转的常见实现（示例规则）

在对话里采用的清晰表达是：

- **180° = mirror X + mirror Y**
- **不交换 XY**

> 这不是唯一实现，但建议把这类规则写成“板级配置注释”，避免半年后自己也看不懂。

---

## 排查步骤（最小闭环）

1. **先归零**：把 `MIRROR_X/MIRROR_Y/SWAP_XY` 全部设为 0，确认基线（避免带着错误补丁继续叠加）。
2. **只调显示**：让显示方向正确（此时先不追求触摸正确）。
3. **再调触摸**：让触摸映射与显示一致（优先复用同一组配置开关）。
4. **最终验收**：
   - 显示方向正确
   - 触摸点击位置与 UI 对齐（四角点对齐）

---

## 验收标准（可执行）

- 点屏幕左上角/右下角时，LVGL 坐标与 UI 控件响应一致
- 不需要在多个地方重复写“翻转/交换”逻辑（集中在 `lvgl_port.*` 或其调用链）
