# Playbook：用 CustomTkinter + 侧边栏导航重构“一个脚本全功能”的 PC GUI

> 来源：对话蒸馏（`C--Users-ChenShuang-Desktop-esp32-demo6.md`）中 PC 端工具从“纵向堆叠的一屏表单（按钮被挤出）”升级为“侧边栏导航 + 分页内容”的落地过程（Time/Weather/Notify 合并到一个工具）。
>
> 目标：把 PC 端工具做成“只启动一个 Python 文件就够用”，并且在小屏/高 DPI 下仍稳健，不被布局问题拖慢联调。

## 一句话结论

**不要把所有功能堆在一屏纵向滚动；用侧边栏导航把功能拆成 Home/Time/Weather/Notify 等页，窗口保持固定尺寸（例如 720×480），体验和可维护性都会明显提升。**

---

## 库选型（对话中的对比结论）

| 库 | 上手成本 | 视觉水平 | 依赖重量 | 迁移成本 |
|---|---|---|---|---|
| CustomTkinter | 低 | 现代深色、圆角、hover | 轻（纯 Python） | 低（继承 tk） |
| PySide6 | 中高 | 工业级最强 | 重 | 大改 |
| ttkbootstrap | 极低 | 主题美化 | 极轻 | 低 |
| Flet | 中 | Material/Flutter | 中 | 重写 |

对话推荐：**CustomTkinter + 侧边栏导航**，并沿用设备端的深紫 + 青绿配色，形成“端到端一致的视觉语言”。

---

## 布局骨架（建议）

- 左侧 180px 侧边栏
  - 顶部：Logo/项目名
  - 中部：导航按钮（Home / Time / Weather / Notify）
  - 底部：连接状态灯 + 当前设备名/地址
- 右侧内容区：按导航切换单页内容

关键点：**一屏只做一件事**（避免“配置表单感”与纵向溢出）。

---

## 逻辑组织（“UI 皮肤”和“业务代码”分离）

对话中强调的落地方式：

- 业务代码保持不动：`BleClient`、定位、天气拉取、UTF-8 固定长度打包等
- UI 重构主要解决两件事：
  - 状态派生：用 `self._connected` + `self._busy` 派生按钮 enable/disable（统一刷新）
  - 交互闭环：连接/断开时自动停掉“自动推送”，避免无效调用

定时策略（对话中的选择）：

- 自动推送优先用 Tk 原生 `root.after` 做定时，避免把 `asyncio` 直接塞进 Tk 主线程导致复杂度上升。

---

## 验收标准

- 功能集中：Time/Weather/Notify 在一个 GUI 内可用（不需要启动多个脚本）。
- 布局稳定：默认窗口尺寸下关键按钮始终可见（不依赖用户手动拉伸）。
- 视觉一致：PC 工具配色/层次与设备端 UI 风格一致，减少“工具像临时脚本”的落差感。
