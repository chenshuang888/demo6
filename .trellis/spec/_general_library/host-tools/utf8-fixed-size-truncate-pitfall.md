## 典型症状

- 文本末尾出现 `�`
- 或固件侧解码失败/显示乱码（取决于解码策略）

---

## 根因

UTF-8 是变长编码。`b = s.encode("utf-8")[:N]` 可能把一个字符的多字节序列切断，导致接收端解码得到替换字符 `�` 或直接失败。

---

## 标准修复：按“最大 N 字节且可正确解码”截断

推荐做法（语言无关）：

1) 先按字节切到 `N`
2) 如果不是合法 UTF-8，就持续删除末尾 1 字节直到合法
3) 再补 `\\0` 到固定长度

Python 参考实现：

```python
def utf8_fixed(s: str, n: int) -> bytes:
    b = s.encode("utf-8")
    b = b[:n]
    while True:
        try:
            b.decode("utf-8")
            break
        except UnicodeDecodeError:
            b = b[:-1]
            if not b:
                break
    return b.ljust(n, b"\0")
```

---

## 验收标准

- 任意中文字符串被截断后：末尾不出现 `�`。
- 固件端 `char[]` 字段始终以 `\\0` 结尾（或至少“确保在 UI 层读取时不会越界”）。
