## 架构卡片（建议放在 README 前半段）

### 1）一句话总结

- 这个项目解决什么问题？（一句话）

### 2）端到端链路（必须）

- 设备侧：输入/输出是什么（传感器/串口/屏幕/存储）
- 主机侧：工具链/服务端做什么（脚本/Web/Electron/云 API）
- 协议：两端怎么对齐（帧格式/字段语义/版本兼容）

### 3）分层与职责（必须）

- 驱动层：外设与引脚/总线
- 适配层：缓冲、调度、协议编解码
- 业务层：状态机、功能模块
- UI 层：页面/渲染/输入

### 4）关键约束（IoT 必填）

- 波特率（串口/USB）
- 采样率/位深/声道（音频）
- 缓冲与内存上限（环形缓冲/队列长度）
- 时序与超时（握手/重试/失败回包）
- 中断/任务优先级（如有）

### 5）如何运行（最小可执行步骤）

- host：安装依赖 + 运行命令 + 验证输出
- device：编译/烧录/monitor + 验证输出
- 参数对齐：波特率/端口号/协议版本/路径

### 6）常见失败与排查入口（至少 3 条）

- 症状 → 可能原因 → 下一步查哪里（文件/日志/命令）

### 7）未知项（允许有，但必须写）

- 未确认的点 + 如何确认（避免 README 编造）
